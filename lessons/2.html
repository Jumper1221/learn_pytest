Курс: Тестирование FastAPI с Pytest

Урок 2: Первые тесты для FastAPI и работа с TestClient

На этом уроке мы напишем первые интеграционные тесты для эндпоинтов FastAPI, используя TestClient. Это позволит нам отправлять HTTP-запросы к нашему приложению и проверять ответы, не запуская реальный веб-сервер.

Шаг 1: Что такое TestClient?

TestClient из fastapi.testclient — это инструмент для тестирования FastAPI приложений. Он построен на базе популярной библиотеки httpx и позволяет:

  • Отправлять запросы к приложению так, как будто оно запущено на сервере, но без реального запуска сервера. Все запросы обрабатываются в памяти, что делает тесты очень быстрыми.
  • Использовать удобный API с методами .get(), .post(), .put(), .delete() и т.д.
  • Автоматически обрабатывать асинхронный код FastAPI в синхронном стиле, что упрощает написание тестов.

Шаг 2: Обновляем наше FastAPI приложение

Давайте добавим несколько новых эндпоинтов в src/main.py, чтобы у нас было больше сценариев для тестирования. На этом этапе мы будем использовать простое "фейковое" хранилище в виде списка, чтобы имитировать работу с данными, не подключая пока настоящую базу данных.

src/main.py
from fastapi import FastAPI, HTTPException, status
from typing import List

app = FastAPI(
    title="My FastAPI Testing App"
)

# Временное хранилище данных для примера (пока без БД)
fake_items_db = [
    {"id": 0, "name": "Foo", "description": "A very nice Foo"},
    {"id": 1, "name": "Bar", "description": "The Bar goes here"},
    {"id": 2, "name": "Baz", "description": "A Baz with no description"}
]

@app.get("/")
def read_root():
    return {"message": "Hello, World!"}

@app.get("/healthcheck")
def healthcheck():
    return {"status": "ok"}

@app.get("/items/", response_model=List[dict])
def read_items(skip: int = 0, limit: int = 10):
    return fake_items_db[skip : skip + limit]

@app.get("/items/{item_id}", response_model=dict)
def read_item(item_id: int):
    # Ищем элемент в нашем фейковом хранилище
    item = next((item for item in fake_items_db if item["id"] == item_id), None)
    if item is None:
        raise HTTPException(
            status_code=status.HTTP_404_NOT_FOUND, 
            detail="Item not found"
        )
    return item

Шаг 3: Создаем файл для тестов API

Создайте новый файл tests/test_api.py. В нем мы настроим TestClient и напишем все тесты для нашего API.

tests/test_api.py
from fastapi.testclient import TestClient
from main import app # Импортируем наше FastAPI приложение

# Создаем экземпляр TestClient, передавая ему наше приложение
# Этот клиент будет использоваться для всех наших тестов
client = TestClient(app)
Напоминание: Импорт from main import app (без src.) работает благодаря настройке pythonpath = src в нашем файле pytest.ini из первого урока.

Шаг 4: Написание тестов для GET-запросов

Теперь добавим в tests/test_api.py тестовые функции для проверки наших эндпоинтов. Каждый тест будет выполнять запрос и проверять результат с помощью assert.

tests/test_api.py (дополнить)
def test_read_root():
    """Тестируем корневой эндпоинт."""
    response = client.get("/")
    assert response.status_code == 200
    assert response.json() == {"message": "Hello, World!"}

def test_healthcheck():
    """Тестируем эндпоинт проверки здоровья."""
    response = client.get("/healthcheck")
    assert response.status_code == 200
    assert response.json() == {"status": "ok"}

def test_read_all_items():
    """Тестируем получение всех элементов."""
    response = client.get("/items/")
    assert response.status_code == 200
    # Проверяем, что ответ является списком и содержит ожидаемое количество элементов
    assert isinstance(response.json(), list)
    assert len(response.json()) == 3 # Изначально в fake_items_db 3 элемента

def test_read_items_with_skip_and_limit():
    """Тестируем получение элементов с параметрами skip и limit."""
    response = client.get("/items/?skip=1&limit=1")
    assert response.status_code == 200
    assert response.json() == [{"id": 1, "name": "Bar", "description": "The Bar goes here"}]

def test_read_existing_item():
    """Тестируем получение существующего элемента по ID."""
    response = client.get("/items/0")
    assert response.status_code == 200
    assert response.json() == {"id": 0, "name": "Foo", "description": "A very nice Foo"}

def test_read_non_existing_item():
    """Тестируем получение несуществующего элемента по ID."""
    response = client.get("/items/999") # ID, которого нет в fake_items_db
    assert response.status_code == 404
    assert response.json() == {"detail": "Item not found"}

def test_read_item_with_invalid_id_type():
    """Тестируем получение элемента с некорректным типом ID."""
    # Вместо числа передаем строку
    response = client.get("/items/abc") 
    # FastAPI (Pydantic) автоматически вернет 422 Unprocessable Entity
    assert response.status_code == 422

Шаг 5: Запуск тестов

Сохраните все изменения и запустите тесты из корневой директории проекта:

Терминал

Вы должны увидеть, что все тесты (и старые из test_sanity.py, и новые из test_api.py) успешно прошли.
