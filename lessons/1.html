Курс: Тестирование FastAPI с Pytest

Урок 1: Основы Pytest и настройка тестового окружения

На этом уроке мы заложим фундамент. Вы познакомитесь с pytest, его основными концепциями и синтаксисом. Мы подготовим структуру проекта и настроим полностью изолированное тестовое окружение с отдельной базой данных PostgreSQL.

Шаг 1: Что такое Pytest и почему именно он?

pytest — это фреймворк для тестирования на Python, который стал стандартом де-факто в сообществе. Его главные преимущества:

  • Простота и лаконичность: Тесты на pytest — это обычные функции Python. Для проверок используется стандартный оператор assert.
  • Мощные фикстуры: Элегантный механизм для управления состоянием тестов (например, подключение к БД).
  • Автоматическое обнаружение тестов: pytest сам находит тестовые файлы (test_*.py) и функции (test_*).
  • Огромная экосистема плагинов: Множество плагинов для расширения функциональности.

Шаг 2: Структура проекта

Правильная организация файлов — ключ к поддерживаемому проекту. Создадим базовую структуру с помощью следующих команд в терминале:

Терминал
mkdir fastapi_testing_project
cd fastapi_testing_project
mkdir src
mkdir tests
touch src/main.py
touch src/config.py
touch tests/test_sanity.py

В итоге у вас должна получиться такая структура:

Структура проекта
fastapi_testing_project/
├── src/
│   ├── main.py
│   └── config.py
└── tests/
    └── test_sanity.py

Шаг 3: Установка зависимостей с Poetry

Мы будем использовать poetry для управления зависимостями. Сначала инициализируем проект:

Терминал
poetry init -n

Теперь добавим основные зависимости для нашего FastAPI сервиса:

Терминал
poetry add "fastapi" "uvicorn[standard]" "sqlalchemy[asyncpg]" "pydantic-settings" "pydantic" "python-dotenv" "asyncpg"

И зависимости, которые нужны только для разработки и тестирования:

Терминал
poetry add --group dev "pytest" "httpx" "pytest-asyncio"

Шаг 4: Создание базового FastAPI приложения

Откройте файл src/main.py и добавьте в него минимальное приложение, чтобы убедиться, что наша базовая настройка работает.

src/main.py
from fastapi import FastAPI

app = FastAPI(
    title="My FastAPI Testing App"
)

@app.get("/")
def read_root():
    return {"message": "Hello, World!"}

@app.get("/healthcheck")
def healthcheck():
    return {"status": "ok"}

Шаг 5: Написание первого "Sanity Check" теста

Теперь напишем наш первый простой тест в файле tests/test_sanity.py. Он не будет проверять логику приложения, а лишь убедится, что pytest в принципе работает.

tests/test_sanity.py
import pytest

def test_example():
    assert 1 == 1

def test_addition():
    assert 2 + 2 == 4

Шаг 6: Запуск тестов

Находясь в корневой директории проекта, выполните в терминале команду:

Терминал
poetry run pytest

Вы должны увидеть примерно следующий вывод, говорящий об успешном прохождении двух тестов.

Шаг 7: Настройка изолированного окружения

Это критически важный шаг. Тесты никогда не должны выполняться на реальной базе данных.

Внимание! Тесты должны быть полностью изолированы. Мы создадим отдельную базу данных и отдельный конфигурационный файл специально для тестов.

1. Создание баз данных

Создайте две базы данных в PostgreSQL:

SQL
CREATE DATABASE app_db;
CREATE DATABASE test_app_db;

2. Конфигурационные файлы

Создайте два файла в корне проекта: .env (для разработки) и .env.test (для тестов). Замените user:password на ваши реальные данные.

.env
DATABASE_URL="postgresql+asyncpg://user:password@localhost:5432/app_db"
.env.test
DATABASE_URL="postgresql+asyncpg://user:password@localhost:5432/test_app_db"

Теперь настроим Pydantic на чтение этих переменных. Откройте src/config.py:

src/config.py
from pydantic_settings import BaseSettings, SettingsConfigDict
import os

# Определяем, какой .env файл использовать
env_file = ".env.test" if os.getenv("TESTING") else ".env"

class Settings(BaseSettings):
    DATABASE_URL: str

    model_config = SettingsConfigDict(env_file=env_file, extra='ignore')

settings = Settings()
Примечание: Мы добавили небольшую логику для автоматического выбора .env.test, если будет установлена переменная окружения TESTING. Pytest будет делать это за нас.

3. Конфигурация Pytest

В корне проекта создайте файл pytest.ini. Этот файл содержит финальные, исправленные настройки, которые решают проблемы с импортами и делают вывод тестов подробным.

pytest.ini
[pytest]
minversion = 6.0
addopts = -ra -v
testpaths = tests
python_files = test_*.py
pythonpath = src
env =
    TESTING=True
  • addopts = -ra -v: Включает подробный (verbose) режим вывода.
  • pythonpath = src: Ключевая настройка! Говорит Pytest, что папка src является корневой для импортов, решая проблему ModuleNotFoundError.
  • env = TESTING=True: Устанавливает переменную окружения, чтобы наш config.py автоматически подхватил настройки из .env.test.
